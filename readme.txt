1. ОБЩАЯ ИНФОРМАЦИЯ
--------------------------------------------------------------------------------
Msudle — образовательная веб-платформа (LMS, аналог Moodle) на Django 4.2.18.
Проект создан для управления учебным процессом: структура вуза, курсы, студенты.
Язык интерфейса: русский (ru), часовой пояс: Europe/Moscow.
Git-репозиторий: https://github.com/stanislaumich/django.git

Стек: Python 3 + Django 4.2 + SQLite + Bootstrap 5 (локально) + Bootstrap Icons.

3. ПРИЛОЖЕНИЯ
--------------------------------------------------------------------------------

3.1. main — лендинг и личные кабинеты.
    - index() — лендинг для гостей и студентов.
    - home() — главная страница сотрудника: блок «Непроверенные задания» (карточки курсов
      с количеством непроверенных ответов, кнопка «Проверить» → таблица успеваемости),
      затем последние объявления по всем доступным курсам.
    - student_home() — сводная успеваемость по всем курсам студента + объявления.
    - dashboard() (/courses/) — личный кабинет (студента/сотрудника/админа):
        * Для сотрудников: плашки курсов в 3 колонки (Bootstrap grid: col-lg-4 col-md-6 col-12).
          На карточке: полное название курса, полное название кафедры,
          уровень доступа (текстом), бейдж непроверенных ответов (если есть),
          две кнопки: «Просмотр» (синяя) и «Редактирование» (красная).
          Права собираются из CourseUserPermission (персональные) и
          CourseGroupPermission (через группы пользователя).
          Курсы сортируются по уровню прав (полный доступ → просмотр).
        * Для студентов: плашки курсов с информацией о кафедре.
    - verbose_name в apps.py: 'ГЛАВНАЯ'.

3.2. structure — University → Faculty → Department.
    - Faculty.dean (FK→User) — декан. Department.head (FK→User) — зав. кафедрой.
    - При назначении/смене декана/зав. кафедрой — автоматическое управление
      группами «Декан»/«Заведующий кафедрой» (auth.Group).
    - __str__() → full_name. identifier: nullable.
    - verbose_name в apps.py: 'ОРГАНИЗАЦИЯ'.
    - Миграции: 0001-0005.

3.3. subject — дисциплины (Subject, FK→Department). __str__() → full_name.
    - verbose_name в apps.py: 'ПРЕДМЕТЫ'.

3.4. students — студенты + аутентификация.
    Модели:
      - StudentGroup (Группа студентов):
          group_number (CharField 50), subgroup_number (PositiveSmallIntegerField, null)
          shifr (FK→umo.Shifr, SET_NULL, null, blank),
          enrollment_year, study_duration_years, study_duration_months,
          faculty (FK→structure.Faculty), education_form (daytime/correspondence)
          unique_together: (group_number, subgroup_number)
      - Student (Студент):
          fio, group (FK→StudentGroup, null, blank), login (unique),
          password (хэш), last_login
          Свойства: is_authenticated, is_active, get_username(), get_full_name()
    Аутентификация: StudentBackend (по login) + EmailOrUsernameBackend (по username/email).
    После логина: сотрудники/админы → /home/, студенты → /student-home/.
    Админка: StudentGroupAdmin (list_display: group_number, subgroup_number, shifr,
             enrollment_year, faculty, education_form), StudentAdmin.
    verbose_name в apps.py: 'СТУДЕНТЫ'.
    Миграции: 0001-0011.

3.5. course — курсы обучения.
     Модели: Course, CourseUserPermission, CourseGroupPermission,
             CourseSection, CourseTopic, LearningUnit, CourseGroupStudent,
             StudentAnswer, CourseAnnouncement, AnnouncementDismiss.
     CourseGroupStudent.group: FK→StudentGroup (вместо group_number).
     Права: edit / create_delete / view / full_access (пользователям и группам).
     При создании курса автоматически назначаются права:
       - создателю — полный доступ (full_access)
       - декану факультета — просмотр (view)
       - заведующему кафедрой — просмотр (view)
       - группе «УМО» — просмотр (view)
       - группе «Ректорат» — просмотр (view)
       - добавляются разделы по умолчанию (6 разделов)
     Управление курсами (администратор):
       - Создание курса: кнопка «Создать курс» в dashboard, форма /course/create/
       - Удаление курса: кнопка «Удалить курс» на странице редактирования
       - При создании/удалении обрабатываются все связанные файлы
     Успеваемость:
       - Таблица /course/<id>/grades/ — группы → студенты → контрольные единицы
       - Модальное окно проверки ответа: дата ответа, дата изменения ответа,
         дата проверки, дата изменения оценки, текущий балл/зачёт, форма оценки
         (баллы 0–max_score или зачёт/незачёт) с кнопкой «Сохранить оценку»
       - Таблица /course/<id>/student_grades/ — успеваемость одного студента
         по курсу (сумма + средний балл)
     Редактирование курса /course/<id>/edit/:
       - Отображение max_score для контрольных единиц в списке
       - Модалка добавления единицы: кнопка «Добавить и далее» с автоинкрементом
         названия (Лекция 1 → Лекция 2), сохранением типа единицы,
         способа оценивания и max_score
     Объявления:
       - CourseAnnouncement: привязано к курсу, автор (User), текст, дата публикации
       - AnnouncementDismiss: персональное скрытие объявления (студент или User)
       - Объявления отображаются на домашних страницах сотрудников и студентов
       - История объявлений доступна на странице просмотра курса (нулевой блок)
       - Скрытие индивидуальное: скрывает только для нажавшего, можно отменить
     Шаблоны:
       - templates/course/create.html — форма создания курса
       - templates/course/edit.html — редактирование (разделы/темы/единицы/группы)
       - templates/course/view.html — просмотр курса (студент/сотрудник)
       - templates/course/grades.html — таблица успеваемости + модалка проверки
       - templates/course/student_grades.html — успеваемость студента
     verbose_name в apps.py: 'КУРСЫ'.
     Миграции: 0001-0017.

3.6. umo — шифры направлений/специальностей.
    Модель Shifr:
        code (CharField 20) — шифр, напр. 6-05-0612-01, 1-40-01-01
        name (CharField 300, null, blank) — название шифра
        qualification (CharField 300, null, blank) — квалификация
    StudentGroup.shifr — FK → Shifr (SET_NULL).
    Админка: ShifrAdmin (list_display: code, name, qualification; search: code, name).
    verbose_name в apps.py: 'СПЕЦИАЛЬНОСТИ'.
    Миграции: 0001-0002.

3.7. accounts — API генерации логина + JS для админки User.
    POST /accounts/generate-login/ — генерирует username из ФИО (транслит + 3 цифры).
    auto_login.js — автозаполнение #id_username при вводе ФИО.
    CustomUserAdmin — кастомный autocomplete с «Фамилия Имя (username)».
    verbose_name в apps.py: 'ПОЛЬЗОВАТЕЛИ'.

3.8. chat — система чатов между преподавателем и студентом.
    Модели:
      - ChatRoom: одна комната на пару (курс, студент).
        unique_together: (course, student).
        unread_count_for(user) — количество непрочитанных сообщений.
      - ChatMessage: текст сообщения, отправитель (sender_student или sender_user),
        is_read, created_at.
    Маршруты:
      - /chat/ — список чатов пользователя (chat:list)
      - /chat/<room_id>/ — комната чата (chat:room)
      - /chat/start/<course_id>/<student_id>/ — создать/открыть чат (chat:start)
    Доступ:
      - Студент видит чаты только по своим курсам.
      - Преподаватель видит чаты по курсам с правами (view и выше).
      - Создавать чаты могут преподаватели с правами edit и выше.
    При входе в комнату сообщения противоположной стороны помечаются прочитанными.
    Ссылка «Чаты» в navbar для авторизованных пользователей.
    verbose_name в apps.py: 'ЧАТЫ'.
    Миграции: 0001.

4. СИСТЕМА АУТЕНТИФИКАЦИИ
--------------------------------------------------------------------------------
 Два независимых бэкенда: StudentBackend + EmailOrUsernameBackend.
 Отображение в navbar:
  - Студент: ФИО, группа/подгруппа

  - Сотрудники/админы → /home/ (главная с объявлениями)
  - Студенты → /student-home/ (сводная успеваемость + объявления)
 Главная страница (/) — лендинг для гостей и студентов.
 Навигация: «Главная» → /home/ (сотрудник) или /student-home/ (студент),
 «Курсы» → /courses/, «Чаты» → /chat/.

9. ИЗМЕНЕНИЯ (лето 2026)
--------------------------------------------------------------------------------
9.1. Оценивание — реальный max_score вместо хардкода 0–100:
     - GRADING_TYPE_CHOICES: «Баллы (0–100)» → «Баллы» (LearningUnit + StudentAnswer)
     - Радио-кнопки в edit.html: «Баллы (0–100)» → «Баллы»
     - Модалка оценки (grades.html) уже динамически использует unit.max_score
     - Валидация в check_answer: max_score = unit.max_score or 100

9.2. Средний балл в таблицах успеваемости:
     - course_grades: каждому студенту добавлен avg_score =
       round(total_score / кол-во_контрольных_единиц, 1)
     - student_grades: также avg_score
     - grades.html: столбец «Σ баллов» → «Σ баллов / Сумма / Средний»,
       ячейка: total_score + avg_score мелким
     - student_grades.html: то же самое

9.3. Двухстрочные заголовки единиц:
     - CSS .two-line-header: word-break / hyphens / max-width: 70px
       (grades.html, student_grades.html, student_home.html)

9.4. Сводная успеваемость на главной странице студента:
     - main/views.py student_home: собирает все курсы с контрольными единицами
       и ответами студента, считает сумму и средний балл
     - templates/main/student_home.html: для каждого курса карточка+таблица
       (как в student_grades.html). Клик на ячейку — модалка с деталями
       ответа и формой отправки/обновления.
     - Заголовок: «Сводная успеваемость / По всем курсам»
     - Шапка карточки: полное название курса (full_name), кнопка «Таблица»
     - Убрана колонка «Студент» (ФИО) из таблиц

9.5. Студенческая модалка ответа:
     - student_grades.html: модальное окно с просмотром предыдущего ответа
       (даты, текст, файл, текущая оценка) и формой отправки/обновления
       (файл + текст). Отправляет POST на course:student_answer.

9.6. Авто-скрытие уведомлений (toasts):
     - base.html: alert-сообщения автоматически закрываются через 5 секунд
       (bootstrap.Alert.getOrCreateInstance().close())

9.7. Таблица успеваемости студента (student_grades.html):
     - Убрана колонка «Студент» из заголовка и строки данных
     - В заголовке страницы: полное название курса (full_name) без кафедры

9.8. Система чатов (приложение chat):
     - ChatRoom: одна комната на пару (курс, студент), unique_together
     - ChatMessage: текст, отправитель-студент или отправитель-преподаватель,
       флаг прочитано, дата отправки
     - Автоматическое создание комнат при заходе студента в список чатов
     - Маршруты: список (/chat/), комната (/chat/<id>/), старт (/chat/start/...)
     - Ссылка «Чаты» в navbar для всех авторизованных пользователей

9.9. Система объявлений (CourseAnnouncement + AnnouncementDismiss):
     - Преподаватели с правами создают объявления (зелёная кнопка на карточке курса)
     - Объявления отображаются на /home/ (сотрудники) и /student-home/ (студенты)
     - Индивидуальное скрытие: каждый пользователь скрывает только для себя
     - История объявлений — на странице просмотра курса (нулевой блок)
     - Можно отменить скрытие в истории

9.10. Домашняя страница сотрудника (main/home.html):
      - Отображает последние объявления по всем доступным курсам
      - Маршрут: /home/ (main:home)

9.11. Подвал (footer):
      - Добавлен в base.html: тёмный фон, одна строка, копирайт с текущим годом

9.12. Рефакторинг главной страницы:
      - index() больше не делает редирект для сотрудников — всегда лендинг
      - Навигация: «Главная» ведёт на /home/ (сотрудник) или /student-home/ (студент)
      - После логина сотрудники → /home/, студенты → /student-home/

9.13. Бейдж непроверенных ответов в dashboard преподавателя:
      - В main/views.py dashboard() для каждого курса подсчитывается количество
        непроверенных ответов (StudentAnswer.checked=False) одним агрегирующим запросом
      - Учитываются единицы как через тему (topic__section__course), так и напрямую
        привязанные к разделу (section__course)
      - В карточке курса (dashboard.html) отображается жёлтый бейдж
        «⚠ Не проверено: N» если количество > 0
      - Для студентов бейдж не отображается

9.14. Блок непроверенных заданий на главной странице сотрудника:
      - В main/views.py home() добавлен подсчёт непроверенных ответов по всем курсам
      - Формируется список unchecked_courses с id, названием, кафедрой и количеством
      - Сортировка: сначала курсы с наибольшим количеством непроверенных
      - В home.html отображается блок «Непроверенные задания» с карточками курсов
      - Каждая карточка: название курса, кафедра, бейдж с количеством, кнопка «Проверить»
      - Кнопка «Проверить» ведёт на /course/<id>/grades/ (таблица успеваемости)
      - Блок отображается только если есть курсы с непроверенными ответами
